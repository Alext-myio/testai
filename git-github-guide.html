Git и GitHub — Краткое руководство

Основные команды Git

  • git init — инициализация репозитория
  • git clone URL — клонирование репозитория
  • git status — проверить статус
  • git add файл — добавить файл в индекс
  • git add . — добавить все изменённые файлы
  • git commit -m "сообщение" — создать коммит
  • git push — отправить на удалённый репозиторий
  • git pull — получить изменения
  • git log — история коммитов
  • git diff — показать изменения

Работа с ветками

  • git branch — список веток
  • git branch имя — создать ветку
  • git checkout имя — переключиться на ветку
  • git checkout -b имя — создать и переключиться
  • git merge имя — слить ветку

Отмена изменений

  • git checkout -- файл — отменить изменения в файле
  • git reset --hard HEAD~1 — откатиться на 1 коммит назад

Работа с GitHub

  1. Создать аккаунт на github.com
  2. Создать новый репозиторий на сайте
  3. Связать локальный репозиторий с удалённым: 
    git remote add origin https://github.com/пользователь/репозиторий.git
  4. Отправить код: git push -u origin master

Полезные советы

  • Всегда пишите понятные сообщения коммитов
  • Делайте коммиты часто и с небольшими изменениями
  • Используйте файл .gitignore для исключения файлов
  • Перед пушем делайте git pull
